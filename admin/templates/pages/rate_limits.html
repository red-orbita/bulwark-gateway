How rate limiting works:

  • Global settings control the default RPM and burst for all tenants.
  • Per-tenant overrides take priority over the global default.
  • Changes are synced to proxy pods via Redis within 5 seconds.
  • Rate limiting uses a 1-minute sliding window (not fixed buckets).
  • Two layers: per-IP (global RPM) + per-tenant (overridable).
