Scanner Pipeline Overview:

  • ML Models: ONNX-based neural networks (DeBERTa, RoBERTa). Require model downloads and numpy/onnxruntime/tokenizers dependencies.
  • RAG Guardrails: Protect against memory manipulation, indirect injection in retrieved chunks, and multi-turn context attacks. Pattern-based, no model needed.
  • Multilingual: Language detection and attack patterns in 10 languages. Enforces allowed_languages policy per agent.
  • Async mode (default): Scanners run in background after request is forwarded. Detections logged as security events but don't block.
  • Blocking mode: Scanners run in hot path before forwarding. Adds latency but can block requests.
  • ML thresholds control sensitivity: higher = fewer false positives, lower = more aggressive detection.
  • Changes are synced via Redis to all proxy pods within seconds.
